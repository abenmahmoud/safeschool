1 Base de donnees Supabase — Passage a l'echelle

CRITIQUE A FAIRE

Pourquoi : Pour supporter des millions d'utilisateurs, la base Supabase gratuite ne suffira pas. Le plan Free est limite a 500 Mo de stockage et 2 Go de bande passante.

Etape 1 : Passer au plan Pro Supabase

  1. Connectez-vous a dashboard.supabase.com
  2. Selectionnez votre projet SafeSchool
  3. Allez dans Settings > Billing
  4. Passez au Plan Pro (25$/mois) ou Enterprise

Etape 2 : Activer le connection pooling (PgBouncer)

  1. Dans le dashboard Supabase, allez dans Settings > Database
  2. Section Connection Pooling, activez PgBouncer
  3. Notez la nouvelle chaine de connexion (port 6543 au lieu de 5432)
  4. Mettez a jour vos variables d'environnement Netlify avec cette nouvelle URL

Etape 3 : Creer les index manquants

Ouvrez l'Editeur SQL dans Supabase et executez :

-- Index pour les requetes frequentes
CREATE INDEX IF NOT EXISTS idx_reports_school_id ON reports(school_id);
CREATE INDEX IF NOT EXISTS idx_reports_tracking_code ON reports(tracking_code);
CREATE INDEX IF NOT EXISTS idx_reports_status ON reports(status);
CREATE INDEX IF NOT EXISTS idx_reports_created_at ON reports(created_at DESC);
CREATE INDEX IF NOT EXISTS idx_reports_school_status ON reports(school_id, status);
CREATE INDEX IF NOT EXISTS idx_audit_logs_school_id ON audit_logs(school_id);
CREATE INDEX IF NOT EXISTS idx_audit_logs_created_at ON audit_logs(created_at DESC);

-- Verifier que les index sont bien crees
SELECT indexname, tablename FROM pg_indexes
WHERE schemaname = 'public'
ORDER BY tablename;

Etape 4 : Activer les replicas en lecture

  1. Dans Settings > Infrastructure (plan Pro)
  2. Activez Read Replicas
  3. Choisissez une region proche de vos utilisateurs (ex: eu-west-1 pour la France)
💡 Conseil : Surveillez les metriques dans Supabase Dashboard > Reports pour identifier les requetes lentes. Ajoutez des index supplementaires si necessaire.

2 Authentification admin avec Supabase Auth + 2FA

CRITIQUE A FAIRE

Pourquoi : L'authentification actuelle utilise un mot de passe encode en base64 (atob(token)). C'est insuffisant pour la production avec des donnees sensibles de mineurs.

Etape 1 : Configurer Supabase Auth

  1. Dans Supabase Dashboard > Authentication > Providers
  2. Activez Email/Password
  3. Activez la verification par email (Email Confirmations)
  4. Dans Authentication > Multi Factor, activez le 2FA/MFA

Etape 2 : Creer la table admin_users

CREATE TABLE admin_users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  auth_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  school_id UUID REFERENCES schools(id),
  email TEXT UNIQUE NOT NULL,
  role TEXT DEFAULT 'admin' CHECK (role IN ('admin','cpe','staff','superadmin')),
  first_name TEXT,
  last_name TEXT,
  created_at TIMESTAMPTZ DEFAULT now(),
  last_login TIMESTAMPTZ
);

-- Securiser avec Row Level Security
ALTER TABLE admin_users ENABLE ROW LEVEL SECURITY;

-- Politique : un admin ne peut voir que les users de son ecole
CREATE POLICY "Admins see own school users" ON admin_users
  FOR SELECT USING (
    school_id = (
      SELECT school_id FROM admin_users WHERE auth_id = auth.uid()
    )
  );

-- Politique : seul le superadmin peut creer des admins
CREATE POLICY "Superadmin can insert" ON admin_users
  FOR INSERT WITH CHECK (
    EXISTS (
      SELECT 1 FROM admin_users
      WHERE auth_id = auth.uid() AND role = 'superadmin'
    )
  );

Etape 3 : Modifier les fonctions Netlify

Remplacez la fonction authCheck() dans chaque fichier .mts :

// AVANT (insecure) :
function authCheck(req: Request): boolean {
  const token = req.headers.get('x-sa-token');
  try { return atob(token) === `${email}:${pass}`; }
  catch { return false; }
}

// APRES (securise avec JWT Supabase) :
import { createClient } from '@supabase/supabase-js';

const supabase = createClient(
  Netlify.env.get('SUPABASE_URL')!,
  Netlify.env.get('SUPABASE_SERVICE_ROLE_KEY')!
);

async function authCheck(req: Request): Promise<{
  authenticated: boolean;
  user?: any;
  role?: string;
}> {
  const token = req.headers.get('Authorization')?.replace('Bearer ', '');
  if (!token) return { authenticated: false };

  const { data: { user }, error } = await supabase.auth.getUser(token);
  if (error || !user) return { authenticated: false };

  const { data: adminUser } = await supabase
    .from('admin_users')
    .select('*')
    .eq('auth_id', user.id)
    .single();

  if (!adminUser) return { authenticated: false };

  return { authenticated: true, user: adminUser, role: adminUser.role };
}

Etape 4 : Modifier le client (index.html / superadmin.html)

  1. Remplacez le stockage localStorage du token base64 par le SDK Supabase Auth
  2. Utilisez supabase.auth.signInWithPassword() pour le login
  3. Utilisez supabase.auth.getSession() pour verifier la session
  4. Ajoutez le header Authorization: Bearer <access_token> a toutes les requetes API
  5. Ajoutez une page de recuperation de mot de passe avec supabase.auth.resetPasswordForEmail()

Etape 5 : Ajouter la dependance Supabase

npm install @supabase/supabase-js

Puis ajoutez la cle SUPABASE_SERVICE_ROLE_KEY dans les variables d'environnement Netlify (Site Settings > Environment Variables). Ne la mettez JAMAIS dans le code client.

⚠️ ATTENTION : La migration doit etre faite progressivement. Gardez l'ancien systeme fonctionnel pendant la transition et testez sur un environnement de staging avant de deployer en production.

3 Chiffrement des donnees sensibles

CRITIQUE A FAIRE

Pourquoi : Les descriptions de harcelement et donnees personnelles doivent etre protegees meme en cas de compromission de la base de donnees. C'est une exigence RGPD pour les donnees de mineurs.

Etape 1 : Verifier le chiffrement au repos

Supabase Plan Pro active automatiquement le chiffrement AES-256 au repos. Verifiez dans Settings > Database que "Encryption at rest" est actif.

Etape 2 : Ajouter le chiffrement au niveau des colonnes

-- Activer l'extension pgcrypto
CREATE EXTENSION IF NOT EXISTS pgcrypto;

-- Ajouter une colonne chiffree pour la description
ALTER TABLE reports ADD COLUMN IF NOT EXISTS description_encrypted BYTEA;

-- Fonction pour chiffrer
CREATE OR REPLACE FUNCTION encrypt_description()
RETURNS TRIGGER AS $$
BEGIN
  IF NEW.description IS NOT NULL THEN
    NEW.description_encrypted = pgp_sym_encrypt(
      NEW.description,
      current_setting('app.encryption_key')
    );
    NEW.description = '***CHIFFRE***';
  END IF;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

-- Trigger automatique a l'insertion
CREATE TRIGGER encrypt_report_description
  BEFORE INSERT ON reports
  FOR EACH ROW
  EXECUTE FUNCTION encrypt_description();

-- Fonction pour dechiffrer (utiliser dans les fonctions serveur)
CREATE OR REPLACE FUNCTION decrypt_description(encrypted_data BYTEA)
RETURNS TEXT AS $$
BEGIN
  RETURN pgp_sym_decrypt(
    encrypted_data,
    current_setting('app.encryption_key')
  );
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

Etape 3 : Configurer la cle de chiffrement

  1. Generez une cle forte : openssl rand -base64 32
  2. Ajoutez-la dans les variables d'environnement Netlify : ENCRYPTION_KEY=votre_cle_ici
  3. Dans Supabase, configurez-la en tant que variable d'environnement de la base
  4. Sauvegardez cette cle dans un gestionnaire de secrets (1Password, Vault, AWS Secrets Manager)

Etape 4 : Chiffrer les donnees existantes

-- Migration des donnees existantes (executez UNE SEULE FOIS)
UPDATE reports
SET description_encrypted = pgp_sym_encrypt(
  description,
  current_setting('app.encryption_key')
),
description = '***CHIFFRE***'
WHERE description_encrypted IS NULL
  AND description IS NOT NULL
  AND description != '***CHIFFRE***';
⚠️ Important : Faites un backup COMPLET de la base AVANT de lancer la migration de chiffrement. En cas d'erreur, les donnees non chiffrees seront perdues si aucun backup n'existe.

// Ajouter cette fonction dans api-notify.mts
async function sendEmail(to: string, subject: string, html: string): Promise<boolean> {
  const apiKey = Netlify.env.get('RESEND_API_KEY');
  if (!apiKey) {
    console.warn('RESEND_API_KEY non configure — email non envoye');
    return false;
  }

  try {
    const res = await fetch('https://api.resend.com/emails', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${apiKey}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        from: Netlify.env.get('EMAIL_FROM') || 'SafeSchool <noreply@safeschool.fr>',
        to: [to],
        subject,
        html
      })
    });

    if (!res.ok) {
      const err = await res.text();
      console.error('Resend error:', err);
      return false;
    }
    return true;
  } catch (e) {
    console.error('Email send failed:', e);
    return false;
  }
}

// Puis remplacez status: 'queued' par un appel reel :

// Configurer VAPID
webpush.setVapidDetails(

  Netlify.env.get('VAPID_PUBLIC_KEY')!,
  Netlify.env.get('VAPID_PRIVATE_KEY')!

// Dans le handler /send :
async function sendPushNotification(subscription: any, payload: any) {
  try {
    await webpush.sendNotification(
      subscription,
      JSON.stringify({
        title: payload.title || 'SafeSchool',
        body: payload.message,
        icon: '/icons/icon-192.svg',
        badge: '/icons/icon-72.svg',
        data: { url: payload.url || '/' }
      })
    );
    return true;
  } catch (error: any) {
    if (error.statusCode === 410) {
      // Subscription expiree — la supprimer
      await removeSubscription(subscription.endpoint);
    }
    return false;
  }
}

8 Audit de securite (Pentest)

CRITIQUE A FAIRE

Pourquoi : Avant la mise en production avec des donnees sensibles de mineurs, un audit professionnel est legalement et ethiquement indispensable.

Societes recommandees en France

SocieteSpecialiteBudget indicatif
YesWeHackBug bounty + pentest5 000 - 15 000 €
SynacktivPentest avance10 000 - 25 000 €
WavestoneAudit complet15 000 - 40 000 €
IntrinsecPentest web5 000 - 12 000 €

Checklist pre-audit

  • Preparer un environnement de staging identique a la production
  • Documenter tous les endpoints API et les roles d'acces
  • Lister les donnees sensibles et leur emplacement
  • Fournir des comptes de test (admin, CPE, eleve)
  • Definir le perimetre : application web, API, base de donnees

Points specifiques a verifier

  1. OWASP Top 10 : injections SQL, XSS, CSRF, broken auth, etc.
  2. RGPD/CNIL : conformite du traitement des donnees de mineurs
  3. Authentification : force brute, session fixation, privilege escalation
  4. API : rate limiting, validation, enumeration des endpoints
  5. Stockage : chiffrement, acces non autorise aux blobs/photos
  6. Service Worker : manipulation du cache, interception
⚠️ Obligatoire : Toutes les vulnerabilites critiques et hautes identifiees doivent etre corrigees AVANT le lancement public. Aucun compromis sur la securite des donnees de mineurs.

10 Scan antivirus des fichiers uploades

MOYENNE A FAIRE

Option A : VirusTotal API (recommandee)

  1. Creez un compte gratuit sur virustotal.com (4 scans/minute gratuits)
  2. Generez une cle API
  3. Ajoutez VIRUSTOTAL_API_KEY dans Netlify
// Ajouter dans api-photos.mts avant le stockage
async function scanFile(fileBuffer: ArrayBuffer): Promise<boolean> {
  const apiKey = Netlify.env.get('VIRUSTOTAL_API_KEY');
  if (!apiKey) return true; // Skip si pas configure

  const formData = new FormData();
  formData.append('file', new Blob([fileBuffer]));

  const res = await fetch('https://www.virustotal.com/api/v3/files', {
    method: 'POST',
    headers: { 'x-apikey': apiKey },
    body: formData
  });

  if (!res.ok) return true; // Fail-open si l'API est down

  const data = await res.json();
  const analysisId = data.data.id;

  // Attendre le resultat (max 30 secondes)
  for (let i = 0; i < 6; i++) {
    await new Promise(r => setTimeout(r, 5000));
    const analysis = await fetch(
      `https://www.virustotal.com/api/v3/analyses/${analysisId}`,
      { headers: { 'x-apikey': apiKey } }
    ).then(r => r.json());

    if (analysis.data.attributes.status === 'completed') {
      const stats = analysis.data.attributes.stats;
      return stats.malicious === 0 && stats.suspicious === 0;
    }
  }
  return true; // Timeout — accept
}

Option B : ClamAV via API cloud

Pour un scan local sans dependance externe, utilisez un service ClamAV heberge (ex: clamav-rest sur Docker).

Etape 2 : Script de test de charge

// load-test.js
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  stages: [
    { duration: '1m', target: 100 },   // Montee progressive
    { duration: '3m', target: 500 },   // Charge soutenue
    { duration: '1m', target: 1000 },  // Pic de charge
    { duration: '2m', target: 0 },     // Descente
  ],
  thresholds: {
    http_req_duration: ['p(95)<500'],  // 95% sous 500ms
    http_req_failed: ['rate<0.01'],     // Moins de 1% d'erreurs
  },
};

const BASE_URL = __ENV.BASE_URL || 'https://votre-staging.netlify.app';

export default function () {
  // Scenario 1 : Consultation de la page d'accueil
  let res = http.get(`${BASE_URL}/`);
  check(res, { 'home OK': (r) => r.status === 200 });

  // Scenario 2 : Soumission de signalement
  res = http.post(`${BASE_URL}/api/lea-stats/push`, JSON.stringify({
    schoolId: 'test-school',
    events: [{ type: 'report_view', timestamp: Date.now() }]
  }), { headers: { 'Content-Type': 'application/json' } });
  check(res, { 'stats push OK': (r) => r.status === 200 });

  // Scenario 3 : Suivi de dossier
  res = http.get(`${BASE_URL}/api/gdpr/export?code=SS-TEST01`);
  check(res, { 'GDPR OK': (r) => r.status !== 500 });

  sleep(1);
}

Etape 3 : Executer sur un environnement de staging

k6 run --env BASE_URL=https://votre-staging.netlify.app load-test.js
⚠️ JAMAIS en production : Executez les tests de charge uniquement sur un environnement de staging. Netlify peut bloquer votre site si vous generez trop de trafic non autorise.

Backups supplementaires

# Script de backup a executer via cron (ex: serveur perso ou GitHub Actions)
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
PGPASSWORD="votre_password" pg_dump \
  -h db.xxxxx.supabase.co \
  -U postgres \
  -d postgres \
  --format=custom \
  --compress=9 \
  -f "backup_safeschool_${DATE}.dump"

# Upload vers un stockage externe (S3, GCS, etc.)
aws s3 cp "backup_safeschool_${DATE}.dump" s3://votre-bucket/backups/

Plan de reprise apres sinistre

  • Tester la restauration d'un backup chaque mois
  • Documenter la procedure de restauration
  • Definir un RPO (Recovery Point Objective) : max 24h de perte de donnees
  • Definir un RTO (Recovery Time Objective) : max 4h de temps de restauration
  • Stocker les backups dans au moins 2 regions geographiques differentes

15 Monitoring avance (Sentry)

HAUTE PARTIEL

Le SDK Sentry a ete integre dans le code (chargement conditionnel). Il reste a :

Etape 1 : Creer un compte Sentry

  1. Inscrivez-vous sur sentry.io (gratuit jusqu'a 5000 erreurs/mois)
  2. Creez un projet de type "JavaScript (Browser)"
  3. Recuperez votre DSN (Data Source Name)

Etape 2 : Activer dans l'application

Ajoutez dans votre code JavaScript (ou via une variable d'environnement injectee) :

// Ajouter AVANT le chargement de l'application
window.SENTRY_DSN = 'https://xxxxxxx@oxxxxxx.ingest.sentry.io/yyyyyyy';

Le code dans index.html detectera automatiquement cette variable et initialisera Sentry.

Etape 3 : Configurer les alertes

  1. Dans Sentry > Alerts, creez une alerte pour les erreurs critiques
  2. Configurez l'envoi par email a l'equipe technique
  3. Ajoutez une alerte si le taux d'erreurs depasse 5% des sessions

Etape 4 : Monitoring Netlify Analytics

En complement de Sentry, activez Netlify Analytics (9$/mois/site) pour :

  • Volume de trafic et pages vues
  • Codes d'erreur HTTP (404, 500, etc.)
  • Bande passante consommee
  • Metriques de performance serveur
